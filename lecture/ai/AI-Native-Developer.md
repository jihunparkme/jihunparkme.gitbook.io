# AI를 '코딩 비서'에서 '개발 파트너'로

## LLM 공략집

### **1️⃣. AI는 '생각'하지 않는다, '계산'할 뿐 (확률적 앵무새)**

LLM의 본질은 **세상에서 가장 정교한 '다음 단어 예측기'**

- **나쁜 요청 (AI의 추측에 의존):**
    > "여기 상품 목록을 보여주는 API 코드가 있는데, 좀 느린 것 같아. 최적화해줘."
- **문제점:** "최적화"라는 단어는 너무 모호. DB 쿼리를 바꾸는 것일지, 캐시를 추가하는 것일지, 알고리즘을 바꾸는 것일지 AI는 '추측'해야만 한다.
- **좋은 요청 (명확한 설계도 제시):**
    
    > "아래 Spring Boot API 코드는 N+1 문제를 발생시켜 성능이 느립니다. Fetch Join을 사용해서 이 문제를 해결하는 방식으로 코드를 수정해줘. JPA Repository와 Service 레이어 코드를 각각 어떻게 바꿔야 하는지 보여줘."


**대응: AI가 추측하게 만들지 말고, 명확한 '설계도'를 그리자.**

원하는 목표, 제약 조건, 해결 방식까지 **유치할 정도로 구체적이고 명확하게** 지시해야 AI는 가장 정확한 계산 결과를 내놓을 수 있다.

### **2️⃣. AI는 '금붕어 기억력'을 가졌다 (Stateless)**

AI가 이전 대화를 모두 기억하는 것처럼 느껴지는 것은 정교하게 구현된 '눈속임'

LLM의 근본적인 아키텍처는 Stateless(무상태). **HTTP 요청**이나 **함수형 프로그래밍의 순수 함수**와 같다. 이전의 호출이 다음 호출에 아무런 영향을 주지 않습니다. 출력은 오직 '현재 입력'에만 의존

우리가 보는 대화창은, 매번 새로운 질문을 보낼 때마다 **이전 대화 기록 전체를 보이지 않는 텍스트로 함께 묶어서** AI에게 통째로 다시 보내주는 것. AI는 이전 대화를 '기억'하는 게 아니라, 매번 '새로 읽는' 것.

**대응: "아까 말했잖아!"는 금물. 모든 요청을 완결된 'API 호출'처럼 만들자.**

매 요청은 그 자체로 완벽한 '작업 지시서(Context)'여야 한다. 필요한 모든 배경 정보, 코드 조각, 설계 의도를 빠짐없이 담아서 전달해야 한다. 이것이 바로 컨텍스트가 전부인 두 번째 이유.

### **3️⃣. AI는 '그럴듯한 거짓말'의 대가 (Hallucination)**

AI의 최우선 목표는 '사실'을 말하는 것이 아니라, '가장 그럴듯한 문장'을 만드는 것. 그래서 자신이 모르는 내용에 대해 "모르겠습니다"라고 말하기보다는, 아주 자신감 있는 태도로 그럴듯한 거짓 정보(환각, Hallucination)를 만들어낸다.

**대응: "믿지만, 반드시 확인한다"를 가슴에 새기자.**

1. **AI는 동료일 뿐, 책임은 나에게:** AI가 짜준 코드는 내가 직접 작성한 코드보다 더 꼼꼼하게 리뷰하고 테스트 하자.
2. **공식 문서와 교차 검증:** 외부 라이브러리나 프레임워크 API를 사용할 때는, 반드시 공식 문서를 옆에 띄워놓고 메서드 이름, 파라미터 등이 실제로 존재하는지 확인하는 습관을 들이자.
3. **'Grounding'으로 거짓말할 공간 없애기:** 프롬프트에 공식 문서의 링크나 정확한 에러 로그 메시지를 함께 제공하면, AI가 기댈 수 있는 '사실'의 땅(Ground)이 생겨 거짓말할 확률이 줄어든다.

## 컨텍스트 관리

> [The New Skill in AI is Not Prompting, It's Context Engineering](https://www.philschmid.de/context-engineering)

에이전트 시스템에서 성공적으로 작동하는지 여부는 작업 메모리에 어떤 정보가 포함될지에 의해 크게 좌우  
대부분의 AI 에이전트 실패는 모델이 아니라 컨텍스트 설계 실패의 결과

✅ **컨텍스트의 구성 요소**

- **시스템 프롬프트/지침:** 모델의 행동을 정의하는 기본 지침과 예시, 규칙 등
- **유저 프롬프트:** 사용자의 즉각적인 요청이나 질문
- **상태/대화 히스토리:** 현재까지의 대화 흐름 및 맥락 정보
- **장기 기억:** 여러 단계를 거친 이전 대화, 사용자 선호도, 과거 프로젝트 요약, 모델이 장기적으로 기억하도록 학습된 정보 모음
- **RAG(검색 기반 증강):** 외부 문서, 데이터베이스, API 등에서 가져온 최신의 관련성 높은 정보
- **사용 가능한 도구:** 모델이 호출할 수 있는 함수, 내장 툴들의 정의 (예: check_inventory, send_email 등)
- **구조화된 출력:** 모델이 따라야 할 응답 형식 정의 (예: JSON)

✅ **컨텍스트 활용법**

1. 텍스트 & 코드 컨텍스트: 가장 기본적이고 강력한 '설계도'
2. 파일 & 문서 컨텍스트: AI에게 '참고 도서'를 통째로 주기
3. 이미지 컨텍스트: 백 마디 말보다 한 장의 '시각 자료'

✅ **건초더미 관리 전략**

1. 건초더미 태우기 - 주기적인 '요약'으로 컨텍스트 압축하기
2. 바늘에 GPS 달기 - `claude.md`와 메모리로 핵심 정보 고정하기
3. 새 건초더미에서 시작하기 - 과감한 리셋의 기술 (`/clear`)

## 프롬프트 엔지니어링

✅ **완벽한 프롬프트의 4요소**

1. 페르소나 (Persona): 누구의 관점에서 말할 것인가?
     - AI에게 특정 '역할'이나 '정체성'을 부여
2. 목표 (Objective): 무엇을 원하는가? 명확한 과업 정의
      - 누가 봐도 오해의 소지가 없도록 명확하고 구체적으로 정의
3. 형식 (Format): 어떤 형태로 결과물을 원하는가?
      - AI가 생성한 결과물을 내가 다시 복사해서 정리하고 있다면, AI를 절반밖에 활용하지 못하는 것
4. 제약조건 (Constraints): 무엇을 '하지 말아야' 하는가?
      - 때로는 '무엇을 하지 말라'는 제약조건이 AI를 올바른 길로 인도하는 데 더 효과적

## 프롬프트 디자인 패턴

1️⃣. 템플릿 제공' 패턴 (Few-shot Prompting)
- 우리가 원하는 결과물의 형식과 스타일을 보여주는 2~3개의 '모범 예시(Template)'를 먼저 제공하는 기술
- 결과물의 일관된 스타일이 중요할 때 유용

**prompt**

```markdown
지금부터 사용자의 자연어 요청을 우리 시스템의 API가 이해할 수 있는 JSON 쿼리로 변환할 거야. 아래 예시들을 보고 숨겨진 패턴까지 완벽하게 학습해줘.  

예시 1 (템플릿 A)
요청: "최근 일주일 동안 '야경' 태그가 달린 서울의 명소 중에서 가장 인기 많은 곳 3개만 찾아줘."
변환: { "city": "서울", "tags": ["야경"], "period_days": 7, "sort_by": "popularity", "limit": 3 }

예시 2 (템플릿 B) 
요청: "부산의 모든 해변 목록" 
변환: { "city": "부산", "tags": ["해변"], "period_days": null, "sort_by": "latest", "limit": null } 
``` 

2️⃣. '단계적 분해' 패턴 (Chain of Thought)
- 복잡한 문제에 대해 AI에게 답을 요구할 때, 프롬프트 끝에 "단계별로 차근차근 생각해 보자." 라는 문장을 추가하여, AI가 자신의 생각의 흐름을 스스로 설명하게 만드는 기술
- 복잡한 요청 시 유용

**prompt**

```text
"이 함수는 배열에서 중복된 숫자를 찾아야 하는데, ... 아래 코드를 보고, 단계별로 차근차근 생각해서 문제의 원인을 진단하고 해결책을 제시해줘. (코드...)"
```

## claude.md & Commands

> `claude.md`: 우리 프로젝트의 '헌법'이자 모든 팀원이 따라야 할 규칙을 담은 문서
>
> `Commands`: 반복적인 업무를 빠르고 일관되게 지시하기 위한 표준 '업무 지시서'

🎉 **프로젝트에서 claude.md와 Commands 자동 생성하기**

- AI에게 '규칙 제정' 명령하기

    ```markdown
    너는 우리 팀의 리드 아키텍트(Lead Architect)야. 방금 `/init`을 통해 이 포트폴리오 프로젝트를 분석했어.

    기존의 피상적인 `claude.md` 파일은 무시하고, 너의 분석을 바탕으로 이 프로젝트의 **'헌법'이 될 상세한 `claude.md` 파일을 처음부터 다시 작성해줘.**

    **작성 가이드라인:**

    1. **`# Tech Stack`**: `package.json` 파일을 분석해서 사용된 핵심 프레임워크, 언어, 라이브러리 버전을 명시해줘. (Next.js, Tailwind, MDX 등)
    2. **`# Content & Data Structure`**: 이 프로젝트는 **콘텐츠를 어떻게 관리하는지** 분석해줘. `content/blog` 폴더 안의 `.mdx` 파일들을 직접 읽어보고, 글의 메타데이터(frontmatter: `title`, `summary`, `publishedAt` 등)가 어떤 구조로 되어있는지 설명해줘.
    3. **`# Code Style & Conventions`**: `app/blog` 와 `components` 폴더 안의 실제 `.tsx` 파일들을 읽고, 이 프로젝트가 따르는 암묵적인 코딩 스타일과 컨벤션을 추론해서 명시해줘.
        - 컴포넌트 정의 방식 (화살표 함수 등)
        - 스타일링 방식 (Tailwind CSS 클래스 사용법, `prose` 클래스 활용 등)
        - 데이터 렌더링 방식 (MDX 콘텐츠를 어떻게 파싱하고 화면에 표시하는지)
    4. **`# SEO Principles`**: 이 프로젝트가 SEO를 위해 어떤 장치들을 사용하고 있는지 분석해줘. (`sitemap.ts`, `robots.ts`, 동적 OG 이미지 생성 등)

    이 모든 내용을 종합해서, 새로운 팀원이 이 문서만 읽어도 프로젝트에 바로 기여할 수 있을 정도로 명확하고 상세하게 작성해줘.
    ```

🎉 **동적 업데이트(#) 활용하기**

- 코딩 세션 중에 새로운 규칙이 생기면, # 기호를 사용해 즉시 업데이트

    ```bash
    # Code Style 섹션에 '모든 새 UI 요소는 MUI 컴포넌트를 사용한다'는 규칙을 추가해줘.
    ```

🎉 **AI가 만든 규칙으로 '자동화 도구' 제작 및 활용**

- 헌법을 따르는 '자동화 도구(Commands)' 생성

    ```markdown
    @claude.md

    너는 이 프로젝트의 콘텐츠 매니저야.
    "$ARGUMENTS"라는 제목의 새로운 포트폴리오 항목을 생성하는 작업을 도와줘.

    **반드시 `@claude.md`에 명시된 모든 기술 스택, 콘텐츠 구조, 데이터 형식 및 코드 스타일 원칙을 완벽하게 준수해야 해.**
    특히, 기존 블로그 글(`content/blog/*.mdx`)과 동일한 방식으로, **MDX 파일의 메타데이터(frontmatter)**를 사용하는 콘텐츠 파일을 생성해줘.

    **요구사항:**
    1.  새로운 콘텐츠를 저장할 `content/portfolio` 디렉토리를 생성하는 명령어를 먼저 제안해줘.
    2.  `content/portfolio` 폴더 안에 `$ARGUMENTS.mdx` 라는 이름으로 파일을 생성해줘.
    3.  파일의 메타데이터에는 `title`, `summary`, `publishedAt`, 그리고 `projectUrl` 필드가 포함되어야 해.
    4.  파일의 본문에는 "여기에 프로젝트 상세 설명을 작성하세요." 라는 플레이스홀더 텍스트를 넣어줘.
    ```

## Claude Hooks

> Hooks는 AI의 모든 행동을 감시하고, 우리가 정한 규칙을 위반할 경우 그 행동을 막거나 수정하도록 피드백을 주는 '자동화된 내부 감사관'이자 '보안 게이트'

**도구 관련 Hooks**
- **`PreToolUse` (사전 실행 훅 / 게이트키퍼):** 
  - Claude가 파일을 읽거나, 코드를 수정하거나 등 어떤 도구를 사용하기 **'직전'**에 실행
  - AI의 행동을 미리 검사하고, 위험하다면 **실행 자체를 '차단'**할 수 있는 훅
- **`PostToolUse` (사후 실행 훅 / 리뷰어):** 
  - Claude가 파일 수정 같은 작업을 **마친 '직후'**에 실행
  - AI가 완료한 작업의 결과물을 검토하고, "코드는 잘 수정했는데, 타입 에러가 발생했어. 이것도 마저 수정해줘." 와 같이 **추가적인 '피드백'**을 AI에게 다시 전송

**세션 및 상호작용 Hooks**
- **`SessionStart`:** 
  - Claude 세션이 시작될 때마다 실행
  - `npm install`을 실행하거나, 환경 변수를 로드하는 등 개발 환경을 초기화할 때 유용
- **`UserPromptSubmit`:** 
  - 우리가 프롬프트를 입력하고 엔터를 누르는 순간 실행
  - 사용자의 입력에 오타가 있거나, 금지된 단어가 포함되어 있는지 검사
